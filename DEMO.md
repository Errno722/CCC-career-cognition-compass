# Demo：从混乱输入到下一步行动

这个 demo 使用完全虚构、已脱敏的求职场景，展示 CCC 如何把混乱信息整理成少量可行动的输出。CCC 是 Career Cognition Compass 的缩写。

## 场景

使用者长期 Gap，之前做过运营相关工作，自学过 AI 工具，但不知道应该继续找运营、转产品，还是尝试 AI 相关岗位。使用者每天只有 2 小时求职时间，并且被社交媒体上的求职建议影响得很焦虑。

## 用户输入

```text
我现在很乱，先不要帮我写简历。

我之前在一家互联网公司做过运营，主要是活动、内容、用户社群这些。
后来因为一些个人原因 Gap 了一年多。
这段时间学过一点 AI 工具，也试过写 prompt、做自动化表格，但都不系统。

我不知道自己应该继续找运营，还是转产品，或者找 AI 相关岗位。
每天大概能花 2 小时求职。
刷小红书、公众号、LinkedIn 越看越焦虑，感觉每个方向都要学很多。

我说 PM 的时候也不确定是产品经理还是项目经理。
希望你先帮我理清楚，不要一上来生成简历。
如果短期找不到全职，我也想知道要不要先找兼职过渡，但不知道找什么。
```

## 输出示例

### 1. 我听到的重点

- 你现在不是单纯缺一份简历，而是需要先判断方向。
- 你的已有经历主要集中在运营、内容、社群、活动和用户触达。
- 你对 AI 工具有初步接触，但目前还缺少可证明的项目证据。
- Gap 超过一年，需要处理求职节奏和信心恢复，但不必把 Gap 变成所有材料的中心。
- 你每天可用时间约 2 小时，因此不适合同时准备太多方向。
- `PM` 目前含义不明确，可能是 Product Manager，也可能是 Project Manager，需要先放进术语表。
- 如果现金流或情绪压力已经影响行动，可以考虑低风险兼职/过渡工作，但不能让它吞掉全部求职精力。

### 2. 还缺的关键信息

优先补充这 3 件事即可：

1. 过去运营工作中，有没有能量化或具体描述的成果？例如增长、转化、留存、活动人数、内容产出、社群活跃。
2. 你更能接受哪类工作：继续执行落地、做需求和协调、做数据分析，还是做 AI 工具应用？
3. 你现在最需要的是快速就业，还是允许用 2-4 周先验证方向？
4. 是否有明确现金流压力？如果有，每周最多能投入多少小时做兼职而不影响正式求职？

### 3. 初版硬技能知识库

```text
硬技能知识库
├─ 运营与增长
│  ├─ 已有线索：活动运营、内容运营、社群运营、用户触达
│  ├─ 证据状态：needs_evidence
│  └─ 下一步：补 1 个活动/社群案例，写清目标、动作、结果
├─ 数据与复盘
│  ├─ 已有线索：可能接触过活动数据、内容表现、用户反馈
│  ├─ 证据状态：needs_confirmation
│  └─ 下一步：确认是否用过 Excel、SQL、BI 或后台数据
├─ AI 工具应用
│  ├─ 已有线索：prompt、自动化表格
│  ├─ 证据状态：learning / needs_evidence
│  └─ 下一步：做一个小案例，例如“用 AI 优化内容生产或用户反馈整理”
└─ 产品/项目协作
   ├─ 已有线索：PM 方向待确认
   ├─ 证据状态：needs_context
   └─ 下一步：确认更接近 Product Manager 还是 Project Manager
```

### 4. 初版术语/缩写表

```text
术语/缩写表
├─ PM：Product Manager / Project Manager（needs_confirmation）
├─ AI：这里暂按人工智能工具理解（inference）
└─ Gap：超过 1 年，求职材料中按需解释，不作为全部叙事中心
```

### 5. 当前判断

短期不建议同时准备“运营、产品、AI 岗位”三个完整方向。更稳妥的做法是先用已有运营经历作为底盘，再验证两个相邻方向。其他职业族群先不展开，避免把注意力打散。

```text
方向 A：运营 / 增长 / 社群 / 内容
├─ 优点：离你过往经历最近
├─ 风险：需要补具体成果和复盘证据
└─ 本周验证：整理 1 个运营项目案例

方向 B：AI 工具应用 / AI 运营
├─ 优点：能把运营经验和 AI 学习结合起来
├─ 风险：需要做一个小作品证明，不要只写“会使用 AI”
└─ 本周验证：做 1 个 AI 辅助运营的小案例

方向 C：产品 / 项目 / 业务协作
├─ 优点：可能承接你的协作和需求整理经验
├─ 风险：PM 含义未确认，证据不足
└─ 本周验证：先确认你想要的是产品方向还是项目管理方向
```

### 6. 下一步 3 个动作

每天 2 小时，先做 3 天，不要一次拉满两周计划。

```text
第 1 天：整理 1 个最完整的运营案例
输出：背景、目标、你的动作、结果、复盘

第 2 天：做 1 个 AI 工具应用小案例
输出：用 AI 解决一个运营问题，例如内容选题、用户反馈分类、活动复盘初稿

第 3 天：比较 2 个真实 JD
输出：一个运营 JD + 一个 AI 应用/产品相关 JD，标出硬技能、工具、你已有证据和缺口
```

### 7. 如果需要兼职过渡

先判断兼职的目的，不要随便找一个消耗很大的工作。

```text
兼职/过渡工作目标
├─ 首要目标：现金流 + 恢复日常节奏
├─ 不适合：高强度、长期占用白天、需要押金/培训费、免费试用的工作
├─ 优先尝试
│  ├─ 内容/社群运营兼职
│  ├─ 客服/用户支持兼职
│  ├─ 数据标注或 AI 测试类兼职
│  └─ 每周固定少量班次、可保留面试时间的线下兼职
└─ 本周动作：找 5 个线索，排除高风险机会，联系 2 个
```

筛选标准：

```text
必须满足：不交押金、不培训贷、时间可控、薪资说明清楚
可以接受：与目标岗位不完全相关，但能稳定现金流和节奏
直接排除：免费试用、长期占用求职时间、要求过多隐私信息
```

## Demo 展示的能力

这个 demo 展示的是：

- 先澄清方向，而不是直接生成简历。
- 把混乱经历整理成职业画像、技能证据和行动。
- 用职业族群筛选缩小范围，而不是一次列出很多岗位。
- 对长期 Gap 保持克制，不把它变成全部叙事中心。
- 对 `PM`、`AI` 等缩写做消歧，不直接假设含义。
- 给出少量下一步动作，避免一次输出太多材料。
- 在长期找不到工作或状态波动时，把行动降到足够小。
- 把兼职/过渡工作当作现金流、节奏和证据问题来判断，而不是当成失败。

## 注意

这是展示样例，不是求职结果承诺。真实使用时，需要根据使用者的具体经历、目标岗位、地区、语言、可用时间和风险承受能力调整。
