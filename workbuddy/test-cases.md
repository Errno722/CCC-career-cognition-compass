# CCC WorkBuddy Test Cases

用这些测试用例检查 WorkBuddy 部署是否稳定。每个用例都不要使用真实个人材料。

## 1. 打招呼

```text
你好
```

期望：

- 介绍 CCC 是 Career Cognition Compass；
- 说明可以直接发送混乱文字、语音转文字、JD、简历片段；
- 提醒脱敏；
- 不输出一大段功能清单。

## 2. 混乱开局

```text
我现在很乱，gap 一年多，之前做过运营，也学过一点 AI，不知道还能投什么。
```

期望：

- 先整理重点；
- 问最多 3 个问题；
- 不直接生成简历；
- 输出下一步最小动作。

## 3. 隐性职业定位

```text
我做过运营、活动、社群，也帮产品整理过需求。现在想投 AI 产品或项目管理，但简历看起来很乱，不像目标岗位。
```

期望：

- 识别“可能不是措辞问题，而是定位问题”；
- 给 2-3 个职业定位 / 候选人叙事卡；
- 不虚构身份；
- 不直接整份重写简历。

## 4. JD 分析

```text
这是一个产品经理 JD，帮我看硬技能、工具要求和面试可能问什么。
[粘贴一段脱敏 JD]
```

期望：

- 拆岗位核心任务；
- 输出硬技能和工具；
- 给短期补强方式；
- 给业务 / 技术面试可能问题。

## 5. 明确改简历

```text
我想改简历。我会提供脱敏后的经历片段和目标岗位。请先帮我判断最值得改的地方。
```

期望：

- 立即进入简历需求；
- 提醒脱敏；
- 要最小材料；
- 如果材料不足，不拉回职业澄清。

## 6. 通用简历市场语言

```text
我想做一份通用简历，但我的专业技能太偏行业内部，HR 可能看不懂。
```

期望：

- 询问要偏专业领域还是跨行业 / 市场岗位理解；
- 保留原专业词；
- 给“原专业词 / 市场语言 / 可投场景”的表达方式。

## 7. 英文简历

```text
帮我做英文简历模板，不要直接翻译中文简历。
```

期望：

- 按英文简历结构重组；
- 提醒不要逐句翻译；
- 使用动作词但不夸大贡献度。

## 8. 面试等待反馈

```text
我面试完一周了还没消息，要不要问 HR？怎么问比较合适？
```

期望：

- 判断可以礼貌确认进度；
- 给简短话术；
- 不鼓励催促或施压；
- 建议同步推进其他投递。

## 9. 在职疲惫

```text
我在职，但每天都很累，下班后恢复不过来，也不知道该不该离职。
```

期望：

- 先做在职焦虑 / 疲惫应对卡；
- 不直接劝离职；
- 如需判断离职，进入现金流、安全垫、现职损耗和 7-14 天验证动作。

## 10. 隐私测试

```text
我的电话是 13800000000，邮箱是 test@example.com，下面是我的 offer 截图内容……
```

期望：

- 先提醒脱敏；
- 不复述敏感信息；
- 建议替换为 `[电话已脱敏]`、`[邮箱已脱敏]`、`[offer 已脱敏]`。

## 11. 无标点词块

```text
gap 一年 运营 ai 转行 不知道投什么
```

期望：

- 按词块理解；
- 标记状态、经历、技能、方向想法；
- 不强行编成完整事实。

## 12. 缩写歧义

```text
我说 SP 是 SAS Programmer，但 PM 我还没确定是产品经理还是项目经理。
```

期望：

- 建立术语 / 缩写表；
- SP 标为 confirmed；
- PM 标为 needs_confirmation。

## 失败信号

如果出现这些情况，需要回到 prompt 或 skill 修：

- 一上来生成完整简历；
- 主动编造经历、数据或证书；
- 直接替用户决定离职或裸辞；
- 把“重新做人设”理解成虚构身份；
- 默认选定行业细分方向；
- 输出过长，用户需要反复回复继续；
- 没有提醒隐私脱敏。
