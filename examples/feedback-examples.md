# 反馈示例

这些示例用于说明什么样的反馈更容易帮助项目迭代。请不要提交真实个人信息。

## 有效反馈：清楚说明场景和问题

```text
使用版本：v0.1.0
使用场景：长期 Gap + 转行

有帮助的地方：
它先帮我整理了当前处境，没有一上来让我写简历。职业画像卡比较有用。

不够好的地方：
方向选择里给了 3 个方向，但没有说明哪个更适合先验证。我看完还是不知道明天该做什么。

希望改进：
希望每个方向后面都加一个“本周最小验证动作”。
```

## 有效反馈：指出缩写误解

```text
使用版本：v0.1.0
使用场景：岗位方向判断

问题：
我说 AI 的时候指 Adobe Illustrator，但模型默认理解成人工智能，导致推荐了 AI 产品经理和大模型工具。

希望改进：
遇到 AI、PM、LLM 这类词时先确认语境，尤其是在设计、法律、数据、产品等不同方向之间。
```

## 有效反馈：指出输出负担

```text
使用版本：v0.1.0
使用场景：快速就业

问题：
输出内容太多，有职业画像、技能表、岗位列表、作品集、面试题。我当时只需要明天先做什么。

希望改进：
快速就业模式下先输出 1 个方向 + 3 个动作，不要一次给很多材料。
```

## 有效反馈：跨模型表现

```text
使用版本：v0.1.1
使用模型：Claude / ChatGPT / Gemini / 本地模型
使用场景：通用 prompt 测试

有帮助的地方：
模型能理解先整理再生成材料。

不够好的地方：
模型没有持续维护术语表，第二轮又把 PM 默认当成 Product Manager。

希望改进：
通用 prompt 里强调每轮都要继承术语表。
```

## 不够有效的反馈：太笼统

```text
不好用。
```

可以改成：

```text
使用场景：校招
不好用的地方：它问了太多问题，我不知道该先回答哪个。
希望改进：第一次最多问 3 个问题，并告诉我哪个最重要。
```

## 不适合公开提交的反馈

```text
这里是我的完整简历、电话、邮箱、公司名、项目数据和面试逐字稿……
```

可以改成：

```text
我有 2 段运营经历、1 个数据分析项目和 1 次产品面试。具体公司和数据已脱敏。问题是：模型把我的“参与项目”写成了“主导项目”，这会夸大经历。
```
