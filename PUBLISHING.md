# 发布路线

CCC 目前采用三条线发布：

```text
GitHub 开源仓库
→ 面向开发者、AI 用户和想复用方法的人

Coze / 扣子编程智能体
→ 面向普通用户，提供可直接聊天的入口

公众号
→ 介绍项目、发布入口、收集反馈和赞赏支持
```

## 1. GitHub 开源发布

GitHub 适合放：

```text
README.md
LICENSE
SUPPORT.md
SKILLS.md
PUBLISHING.md
DEMO.md
FEEDBACK.md
prompts/
skills/
coze/
examples/
integrations/wechat-bot/
```

不要上传：

```text
真实简历
真实 JD 的未脱敏截图
offer / 合同 / 薪资截图
完整面试记录
公司内部信息
个人联系方式
平台密钥、Token、后台配置截图
```

发布前检查：

```text
1. README 是否说明 CCC 是什么
2. LICENSE 是否存在
3. SUPPORT 是否说明赞赏是自愿的
4. skills 是否校验通过
5. prompts 是否没有隐私内容
6. 示例是否是虚构或已脱敏
7. 没有残留旧商业化或保证 offer 等措辞
```

## 2. Coze 商店发布

Coze 商店适合发布普通用户可用的 CCC 智能体。

发布前至少测试这些输入：

```text
你好
gap 一年 运营 ai 转行 不知道投什么
我现在很乱，gap 一年多，之前做过运营，也学过一点 AI，不知道还能投什么。
我想转行，但不知道自己的经历能迁移到哪里。
这是一个产品经理 JD，帮我看硬技能、工具要求和面试可能问什么。
我想改简历，先不要做职业澄清。
我之前做断路器相关工作，帮我看看能投什么方向。
我上一份简历是按项目管理 JD 改过的，现在要投研发 JD，帮我避免上一版偏向带偏。
我刚面试完，只记得 SQL、留存、用户分层、dashboard。
我在职，但是每天很累，下班后恢复不过来，也不知道该不该离职。
我现在很想裸辞，没有 offer，也不知道该不该先离职。
```

合格标准：

```text
回复不超过 300 字为主
公众号渠道首屏尽量不超过 180 字
不使用 emoji
不直接生成大报告
语气不生硬、不像固定模板
有状态承接，但不输出大段情绪安慰或鸡汤
无标点碎片输入时会先归类词块，不要求用户重写
明确要求改简历时会马上进入简历修改，不强行职业澄清
不承诺 offer
不编造经历
不直接劝辞或劝留
遇到行业/产品多分支时不自动选择细分方向
切换 JD 或岗位族群时不自动继承上一版简历偏向
首次或高风险场景提醒用户脱敏和自行决策
内容过长时优先给“JD / 简历 / 面试 / 方向”等具体选项
长 JD、完整简历、面试题库会先分流，不一次性展开
```

Coze 商店文案见：[coze/STORE_LISTING.md](coze/STORE_LISTING.md)。

Coze 系统提示词见：[coze/SYSTEM_PROMPT.md](coze/SYSTEM_PROMPT.md)。

## 3. 公众号配合使用

公众号适合做：

```text
项目介绍
使用说明
Coze 智能体入口
GitHub 开源地址
反馈入口
赞赏入口
版本更新说明
```

公众号不适合公开：

```text
真实用户案例
未脱敏简历或 JD
后台 token
平台配置截图
过度承诺的宣传语
```

公众号菜单建议：

```text
开始使用
项目说明
反馈/赞赏
```

自动回复建议：

```text
你好，我是 CCC，Career Cognition Compass。

你可以直接发一段很乱的求职情况，比如 Gap、转行、在职很累、看到一个 JD、不知道该投什么，或者面试后只记得几个关键词。

我会注意你的状态，但主要功能不是情绪陪伴或心理咨询，而是帮你把信息、方向和下一步行动理出来。

请先脱敏，不要发送身份证、电话、offer、合同、薪资截图或完整简历。CCC 只提供整理和建议，最终决定仍由你自己做。
```

## 当前优先级

```text
1. GitHub 开源仓库整理到 v0.1
2. Coze 智能体完成核心场景测试
3. 发布到微信公众号订阅号审核
4. 写公众号介绍文章
5. 找 5-10 人小范围试用
6. 根据反馈更新 prompt、skills 和 README
7. 再考虑发布到 Coze 商店
```
