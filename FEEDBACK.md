# 使用反馈

谢谢你愿意反馈 CCC 的使用体验。CCC 是 Career Cognition Compass 的缩写。这个项目更重视“使用后是否更清楚、更能行动”，不追求一次生成很多材料。

## 先保护隐私

请不要在公开 GitHub Issue、Discussion 或评论里提交：

- 真实姓名、电话、邮箱、地址、身份证、护照、签证号码；
- 完整简历、offer、合同、薪资截图；
- 未脱敏的面试记录、公司内部信息、客户信息；
- 任何你不希望被搜索引擎或他人看到的求职细节。

可以这样脱敏：

```text
[姓名]，[城市]，[公司A]，[岗位B]，[项目代称]，[具体数据已脱敏]
```

## 3 分钟反馈

如果你只想快速反馈，请回答这 5 个问题：

```text
使用版本：v0.1.0 / v0.1.1 / v0.1.2 / v0.1.3 / v0.1.4 / v0.1.5 / 不确定
使用场景：长期 Gap / 转行转岗 / 校招 / 在职换工作 / 海外求职 / 快速就业 / 面试复盘 / 其他

1. 使用后，我是否更清楚自己现在的求职状态？
评分：1-5

2. 使用后，我是否知道下一步要做什么？
评分：1-5

3. 哪一部分最有用？
例如：职业画像、方向选择、硬技能知识库、术语/缩写表、简历草稿、面试复盘、情绪稳定、兼职过渡、14 天计划

4. 哪一部分不够好？
例如：太泛、太长、问太多、没理解行业、没有行动感、让我更焦虑、像模板话术

5. 你希望下个版本优先改什么？
```

## 反馈维度

优先反馈这些问题：

- **清晰度**：是否帮你更清楚地理解自己的处境？
- **行动感**：是否给出一个能做的小步骤？
- **真实感**：输出是否像你自己的经历，而不是套话？
- **克制感**：是否避免夸大和过度包装？
- **负担感**：材料是否太多？有没有让你更焦虑？
- **情绪承接**：长期找不到工作、Gap 过长或低能量时，是否帮你找到一个能开始的小动作？
- **过渡支持**：兼职、临时工作或过渡工作建议是否现实，是否避免了高风险机会？
- **准确性**：岗位、行业、硬技能、软件工具判断是否合理？
- **术语理解**：是否正确处理 `SP`、`PM`、`AI`、`LLM` 等缩写？
- **隐私安全**：有没有要求过多敏感信息？
- **跨模型适配**：在 ChatGPT、Claude、Gemini、本地模型或其他 agent 上是否还能正常使用？

## 适合提交到 GitHub 的反馈

适合公开提交：

- 某个流程太长或太短；
- 某个输出格式不好用；
- 某个场景没有覆盖；
- 某个缩写容易误解；
- 某个示例 prompt 不清楚；
- 某个大模型执行时容易跑偏。

不适合公开提交：

- 完整简历；
- 真实面试过程；
- 未脱敏的 JD 或公司信息；
- 个人焦虑、健康、家庭或财务细节。

如果反馈里包含隐私，请先脱敏，或者只描述问题类型。
