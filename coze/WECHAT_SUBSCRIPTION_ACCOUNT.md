# Coze 智能体发布到微信订阅号

CCC 的普通用户入口可以采用：

```text
Coze / 扣子编程低代码智能体
→ 发布到微信公众号订阅号
→ 用户在公众号里直接对话
```

## 前提

```text
1. 已创建微信订阅号
2. 已创建扣子编程低代码智能体
3. 已在微信开发者平台找到公众号 AppID
4. 公众号管理员可以扫码授权
```

订阅号首次发布通常需要微信审核。审核通过前，可以先在 Coze 预览区测试智能体。

## 操作顺序

```text
1. 在扣子编程中打开 CCC 智能体
2. 点击部署
3. 进入发布
4. 选择微信公众号（订阅号）
5. 填入公众号 AppID
6. 管理员扫码授权
7. 勾选订阅号渠道
8. 填写发布记录
9. 提交发布
10. 等待微信审核
```

## 公众号菜单建议

```text
开始使用
项目说明
反馈/赞赏
```

## 自动回复建议

```text
你好，我是 CCC，Career Cognition Compass。

你可以直接发一段很乱的求职情况，比如 Gap、转行、在职很累、看到一个 JD、不知道该投什么，或者面试后只记得几个关键词。

我会注意你的状态，但主要功能不是情绪陪伴或心理咨询，而是帮你把信息、方向和下一步行动理出来。

请先脱敏，不要发送身份证、电话、offer、合同、薪资截图或完整简历。CCC 只提供整理和建议，最终决定仍由你自己做。
```

## 真机测试清单

审核通过后，用微信真机测试：

```text
你好
我现在很乱，gap 一年多，不知道还能投什么。
我想转行，但不知道经历能迁移到哪里。
这是一个产品经理 JD，帮我分析。
我刚面试完，只记得 SQL、留存、用户分层。
我在职很累，不知道该不该离职。
```

观察：

```text
回复是否过长
是否触发 15 秒超时
是否提示“继续”
是否提醒脱敏
是否不承诺 offer
是否不直接劝辞
是否能继续追问而不是每轮重新开始
```

## 15 秒超时处理

微信订阅号渠道可能出现类似：

```text
思考中，请回复继续
```

这通常不是内容错误，而是回复生成超过了微信渠道的时间限制。解决思路是让公众号端默认短回复、分步回复。

建议配置：

```text
1. 系统提示词使用“微信订阅号 15 秒快速回复规则”
2. 默认回复控制在 180 字以内
3. JD、简历、面试复盘不要一次性完整展开
4. 复杂任务先给第一屏结果，再让用户回复“JD / 简历 / 面试 / 方向”
5. 用户回复“继续”时，只继续上一块内容的一小段
```

容易超时的场景：

```text
长 JD 一次性分析
完整简历生成
多岗位方向对比
面试题库一次性展开
大段职业规划
```

对应改法：

```text
长 JD → 先给核心任务、硬技能、1 个缺口
完整简历 → 先给简历补丁，不直接生成全文
多方向 → 先给最多 2 个方向
面试题库 → 一次只给 1-2 个题型
职业规划 → 先给今天 1 个动作
```

## 注意

- 公众号里不要公开真实用户案例。
- 不要发布未脱敏简历、JD、面试记录或薪资截图。
- 不要把平台 token、AppID 后台截图、管理员二维码公开。
- 如果回复经常超时，先缩短系统提示词和默认回复长度。
